「Webコンテンツ作成」の成果

本テーマについて

目的: 情報工学の専門科目の基礎知識を応用した実践的な能力の修得.
目標: ①実験の一連の流れの実践　②グループワークの円滑化　③モーション解析・編集の習得　④HTML等を用いたWebページの作成.

プロジェクト課題について

　世界の都市12地点から1つ選んで検索ボタンを押すと, 選んだ都市の軽度, 緯度, 天気, 最低気温, 最高気温, 湿度, 風速, 風向を表示するWebコンテンツを作成した.
　検索にはプルダウンを使用しており, 都市名の英語表記と日本語表記に加えて国名も添えてあるため, 快適に検索できるようになっている. 検索ボタンを押すと, 選んだ都市名に応じたidが選ばれ, データが出力される.
　ページ全体を青空をイメージした青色で作成した. 文字は濃い色を使用し, 反対に背景は薄い色を使用することで, 文字をより鮮明にしている. また, 空行を入れることで文字をより読みやすくしている.
プロジェクト課題で作成したページ : project.html

数当てゲーム

　乱数で 1〜10の値を1つ決めて, 「答え」とする. ユーザは答えを予想して, 数を最大で3回入力できる. ユーザの入力した予想が外れた場合は，入力した数よりも答えが大きいか小さいか表示する. 3回以内に答えの数を当てればユーザの勝ちで, そうでなければ負けとなる.
　テキストボックスに整数を指定して回答ボタンを押すと，正誤判定の結果が表示される. 回答ボタンを押す度に回答回数（予想回数）がカウントされ，ページに表示される. 正解すると正解のメッセージが表示される. 3回間違えると答えが表示される. 4回以上回答すると, ゲームが終わっているというメッセージが表示される.
数当てゲームのページ : kazuate.html

各授業回の学習事項

第1回 WWW と HTML

　HTMLは要素で作られ, タグで最初と最後を囲んだ部分が要素となる. 要素にはブロック要素とインライン要素があり, ブロック要素はインライン要素の中に配置できない. br要素は段落の中で改行を入れる要素で, 閉じタグを持たないから要素として使用する. HTML文書には5つの特殊文字があり, Webページで表示させるためには特別な記述をする必要がある.
演習の成果 : 演習1-4 : HTMLの様々な要素

第2回 CSS による Webページの装飾

　ページを装飾する記述をCSSと言い, HTML文書の要素ごとに様々な装飾を追加できる. 装飾の内容は, 文字や背景の色, フォント, サイズ, 斜体, 下線, 枠線, 空白, 表示方法などの変更や追加がある.
演習の成果 : 演習2-3 : ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

　Javaはコンパイル言語であるが, JavaScriptはインタプリタ言語である. 動的型付けをする言語であるため, 間違ったプログラムを実行すると, コンパイルエラーではなく実行時エラーが発生する. JavaScriptは, 手軽に実行とデバックを繰り返すことができる長所に対し, エラーの原因を見つけるのが難しいという短所がある.
演習の成果 : 演習3-6 : オブジェクトの参照

第4回 DOM と JavaScript

　DOMはWebコンテンツを表現するデータ構造であり, JavaScriptと組み合わせることで動きのあるWebページを作ることができるようになる. テキストボックスやラジオボタンなど様々なユーザインタフェースを表示させることができる.
演習の成果 : 演習4-10 : 足し算ページ

第5回 イベント処理

　DOMを使うことで, 要素の新規作成や編集, 削除をも行うことができる. createElement(要素名)で新規作成, 要素1.insertAdjacentElement('beforeend(またはafterend)', 要素2)で配置, 要素.setAttribute(属性名, 値)で属性の設定, 要素.remove()で削除を行う.
演習の成果 : 演習5-7 : DOMプログラミング(2)

工夫点, 苦心点, 強調点, 感想

　Javascrpit は Java に比べてとても難しく, コードと実行結果を理解するまでにかなり時間がかかった. どうしてもわからなかったことは先輩方に質問して教えていただいたが, できるだけ自力で他のコードと見比べ, 照らし合わせ, 少しずつコードを書き換えては実行してを繰り返していき, なんとか最後までプロジェクト課題を完成させることができた. 完成形と手がかり（資料）を与えてもらい, そこから試行錯誤することがパズルのように感じられ, 作業を楽しむことができた. このような学習方法は自分に向いていると強く感じた.
　プロジェクト課題の検索機能について, select要素を使用してプルダウン形式にした. 都市名の英語表記と日本語表記, その都市がある国名を全て検索で使える方が良いと考え, テキストボックス形式のsearch要素から変更した. search要素からselect要素に変更するのを最初は躊躇していたが, select-sample.htmlとその.jsのおかげでコードと実行結果を理解することができ, スムーズにプルダウン形式に変更することができた. 都合上出力できる都市のデータが限られているため, プルダウン形式にしたことで確認作業もしやすく, 一石二鳥の結果を得られた. 有意義な改良ができたと自負している.
　背景の色と文字の色を決めるのもとても大変だった. 美術分野の知識が乏しいため, 「どの色の組み合わせならば文字を読みやすくできるのか」がわからず, 試行錯誤を繰り返した. 自分の性格上シンプルなものやコンパクトなものが好きなため, より一層Webページの装飾を考えるのには苦労した. 最終的に天気から空の色を連想し, 青系色を使用することに決めた. またmarginやpaddingを使用して行間を作ることで, 一つ一つの項目を見やすくすることができた. 検索欄だけは敢えて色や大きさに変更を加えないことで, タイトルと検索結果との境目を作り, 検索結果をより一層際立たせている. 主観的な感想であるが, 枠線をつけたことでその内側にある文字たちがより目を引くようにできたと感じている. 文字をページの構造上, フルスクリーンで表示すると画面右側がもの寂しくも感じられるが, 画面を半分にすることで寧ろスッキリしていてとても良い仕上がりとなる.
　Webコンテンツの作成というテーマに対して, 最初はあまり強い関心を持っていなかったためとても苦労したが, Javascriptをうまく実行できたときに得られる達成感から楽しさを感じることができた. 今後趣味でブログなどのWebページを作ってみたいと思える良いきっかけになった. 　（1060文字）
